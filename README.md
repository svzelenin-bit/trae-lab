# trae-lab

Учебный репозиторий для обучения вайб-кодингу с **Trae AI + GitHub MCP**.  
Здесь я отрабатываю базовый и практический сценарий работы: идея → задача → план → маленькие изменения → проверка → коммит.  

## Цель репозитория

Этот репозиторий нужен для спокойной учебной практики без риска сломать рабочие проекты.  
Здесь размещаются только тестовые, учебные и экспериментальные проекты.

## Что я хочу освоить

- Понимать структуру проекта, а не просто копировать код.
- Давать ИИ маленькие и понятные задачи.
- Работать через план, а не через хаотичные правки.
- Менять только нужные файлы.
- Проверять результат после каждого шага.
- Делать аккуратные коммиты с понятными названиями.

## Базовый подход к работе

1. Сначала изучить проект и понять, что уже есть.
2. Затем попросить ИИ предложить короткий план из 3 шагов.
3. Не вносить изменения без подтверждения.
4. Менять только те файлы, которые относятся к задаче.
5. После каждого изменения проверять результат.
6. Делать небольшие коммиты, чтобы было легко откатиться назад.

## Первые учебные проекты

Для старта в этом репозитории будут использоваться простые и полезные форматы:

- Лендинг
- Форма заявки
- Простая админ-страница
- Таблица или мини-панель с данными
- Небольшой бизнес-интерфейс

## Правила работы с ИИ

Основные правила вынесены в отдельный файл [`PROJECT_RULES.md`](./PROJECT_RULES.md).  
Этот файл нужен для того, чтобы ИИ работал аккуратно, предсказуемо и без лишнего хаоса.

## Рабочий шаблон запроса к ИИ

Используем базовый формат:

> Проанализируй проект.  
> Предложи план из 3 шагов.  
> Ничего не меняй без моего подтверждения.  
> После подтверждения измени только нужные файлы.  
> Объясняй кратко и по-русски.
